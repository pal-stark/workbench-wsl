# WSL


## Итак, начнём.

## Установите нужную версию wsl.

Рекомендуется использовать версию Ubuntu 26.04 LTS. Для этого выполните команду:
```
wsl --install -d Ubuntu-26.04
```
        
Полный список доступных версий можно посмотреть с помощью команды:

```
wsl --list --online
```

Введите логи н и пароль для создания пользователя в Ubuntu.

Запустите установленную версию Ubuntu с помощью команды:

```
wsl -d Ubuntu-26.04
```

Выполните скрипт установки необходимых пакетов с помощью команды:
```
intall.sh
```

После успешного выполнения скрипта, вы можете использовать WSL для разработки и тестирования ваших приложений.

Выйдите из WSL с помощью команды:
```
exit
```

Войдите в WSL снова с помощью команды:
```
wsl -d Ubuntu-26.04
```

Теперь можно проверить установленные пакеты и их версии с помощью команды:
```
dpkg -l
```

Работу docker в WSL можно проверить с помощью команды:
```
docker --version
```
Или проверить статус службы docker с помощью команды:
```
sudo systemctl status docker
```

Версию php можно проверить с помощью команды:
```
php --version
```

Версию symfony можно проверить с помощью команды:
```
symfony --version
```
Работоспособность composer можно проверить с помощью команды:
```
composer --version
```

В случае успешной установки, можно приступить к разработке и тестированию приложений.

Следуйте инструкциям в документации для настройки вашего проекта и запуска его в WSL.


## Сервисы для удобства локальной раработки.


Настройка и запуск k0s для тестирования развёртывания приложений в kubernetes

```
sudo k0s install controller --single
```

```
sudo k0s start # wait a minute
```

Тестирование запуска и готовности к работе.

```
sudo k0s kubectl get nodes
```
